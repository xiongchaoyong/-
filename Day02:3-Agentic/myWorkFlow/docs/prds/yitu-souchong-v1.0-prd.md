# 以图搜宠 - 产品需求文档 (PRD)

## 需求描述

### 背景
- **业务问题**：领养用户有模糊偏好但缺乏明确选择标准，面对海量宠物信息无从下手
- **目标用户**：有养宠意向但不确定具体选什么宠物的潜在领养者
- **价值主张**：通过图像特征识别，让用户"看到喜欢的就找到相似的"，降低选择门槛

### 功能概述
- **核心功能**：用户上传宠物图片 → 系统自动识别特征 → 推荐相似度最高的待领养宠物
- **功能边界**：
  - 包括：单图/多图上传、自动特征识别、相似度匹配排序、结果展示、历史记录
  - 不包括：视频识别、宠物健康诊断（F005）、品种百科查询
- **用户场景**：用户在网上看到一只喜欢的猫，截图上传，系统匹配出领养库中最相似的待领养宠物

### 详细需求
- **输入**：JPG/PNG/WebP格式图片，单张≤10MB，支持多图上传（必须同一只宠物）
- **输出**：Top 1相似宠物（核心信息卡片 + 相似度百分比 + 跳转详情页入口），低于70%相似度时附带"相似度较低"提示
- **用户交互**：上传 → 前端压缩至2/3 → 后端调用阿里云动物识别API → 特征提取 → 宠物库匹配 → 返回结果
- **边界情况**：
  - 非宠物图片：上传失败，提示"请上传宠物图片"
  - 不支持的格式：提示"请上传 JPG/PNG/WebP 格式图片"
  - API超时/失败：提示"识别服务暂不可用，请稍后重试"
  - 相似度<70%：仍展示结果但提示"相似度较低，建议扩大筛选范围"
  - 宠物库为空：提示"暂无待领养宠物，请稍后再试"

## 设计决策

### 技术方案
- **架构选择**：前端 Vue 3 + 后端（待定）+ 阿里云视觉AI动物识别API
- **关键组件**：
  - 图片上传组件（支持拖拽、多选、格式校验）
  - 前端压缩模块（压缩至原图2/3）
  - 识别服务代理（调用阿里云API + 重试机制）
  - 特征匹配引擎（脸型 > 体型 > 品种，用户可调整权重）
  - 历史记录服务
- **数据存储**：搜索历史表（用户ID、图片URL、匹配结果、相似度、时间戳）
- **接口设计**：
  - `POST /api/search/by-image` — 以图搜宠
  - `GET /api/search/history` — 搜索历史
  - `DELETE /api/search/history/:id` — 删除历史记录

### 约束条件
- **性能要求**：上传到返回结果 ≤ 10秒
- **兼容性**：支持主流浏览器（Chrome/Firefox/Safari/Edge）
- **安全性**：图片上传防恶意文件，前端压缩减少带宽
- **可扩展性**：初期小规模低并发，后续按需扩展

### 风险评估
- **技术风险**：阿里云API识别准确率不稳定 → 设定70%最低阈值，低相似度主动提示
- **依赖风险**：第三方API服务不可用 → 友好降级提示 + 自动重试
- **进度风险**：API集成调试耗时 → 优先完成单图识别，多图作为增量迭代

## 验收标准

### 功能验收
- [ ] 支持JPG/PNG/WebP格式图片上传，单张≤10MB
- [ ] 非宠物图片正确拦截并提示
- [ ] 多图上传后取平均相似度排序，返回Top 1
- [ ] 相似度<70%时展示结果并附带低相似度提示
- [ ] 用户可手动调整匹配权重（脸型/体型/品种）
- [ ] 搜索历史可查看和删除
- [ ] API异常时显示"稍后重试"提示

### 质量标准
- [ ] 上传到结果返回 ≤ 10秒
- [ ] 前端压缩至原图2/3
- [ ] 测试覆盖率 ≥ 80%
- [ ] 无XSS/文件上传安全漏洞

### 用户验收
- [ ] 上传交互流畅，拖拽和多选均可用
- [ ] 结果卡片信息清晰，可一键跳转详情页
- [ ] 错误提示文案友好易懂
