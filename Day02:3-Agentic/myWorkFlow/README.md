# MyWorkFlow — AI辅助需求分析工作流

基于Claude Code的智能体+技能协作系统，用于宠物领养平台的需求分析、澄清和PRD生成。

## 目录结构

```
myWorkFlow/
├── agents/                        # 智能体定义
│   └── product-manager.md         # 需求分析师（产品经理）
├── .claude/                       # 工作流配置
│   ├── CLAUDE.md                  # 项目宪章（会话自动加载）
│   └── skills/                    # 技能定义
│       └── requirements-clarity/
│           └── SKILL.md           # 需求澄清技能（100分评分制）
├── docs/                          # 文档资产
│   ├── 宠物领养调研报告.md         # 市场调研与竞品分析
│   └── prds/                      # 生成的PRD文档
│       └── yitu-souchong-v1.0-prd.md
└── output/                        # 输出产物
```

## 工作流程

```
用户需求 → 需求分析师（Agent）→ 需求澄清（Skill）→ PRD文档
```

### 阶段1：需求输入

用户提供需求材料（调研报告、功能描述、用户反馈等），系统通过 `.claude/CLAUDE.md` 自动加载项目宪章和开发规范。

### 阶段2：智能体路由

根据任务类型匹配合适的智能体：

| 触发条件 | 智能体 | 行为 |
|----------|--------|------|
| 需求分析、PRD编写、功能拆解 | product-manager | 读取agent定义 → 按角色执行需求分析 |

### 阶段3：技能激活

当智能体检测到模糊需求时，自动调用 `requirements-clarity` 技能：

1. **初始评估** — 从4个维度（功能清晰度/技术具体性/实现完整性/业务上下文）评分
2. **交互澄清** — 每轮2-3个问题，逐轮递进
3. **评分迭代** — 每轮更新评分，直到 ≥ 90/100
4. **PRD生成** — 评分达标后输出标准化PRD到 `docs/prds/`

### 阶段4：输出交付

PRD包含6个标准章节：需求描述、设计决策、验收标准、执行阶段、风险评估、技术方案。

## 使用方式

### 启动需求分析

```
分析这个需求，生成PRD文档
```

### 单独澄清需求

```
帮我澄清「以图搜宠」功能的需求
```

### 查看已生成的PRD

PRD输出在 `docs/prds/` 目录，命名格式：`{feature_name}-v{version}-prd.md`

## 评分体系

| 维度 | 满分 | 评估内容 |
|------|------|----------|
| 功能清晰度 | 30 | 输入输出、用户交互、成功标准 |
| 技术具体性 | 25 | 技术栈、集成点、约束条件 |
| 实现完整性 | 25 | 边界情况、错误处理、数据验证 |
| 业务上下文 | 20 | 问题陈述、目标用户、成功指标 |

**合格线：≥ 90分**
