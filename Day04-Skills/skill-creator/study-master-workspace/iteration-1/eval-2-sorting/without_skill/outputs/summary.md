## 2. 三大排序深度剖析

---

### 2.1 快速排序 (Quick Sort)

#### 核心思想

**分治法 (Divide and Conquer)**：选一个基准元素（pivot），将数组分为两部分，左边都小于 pivot，右边都大于 pivot，然后递归地对左右子数组排序。

#### 算法步骤

```
1. 选择 pivot（通常选最后一个、第一个、中间元素，或随机选）
2. partition（分区）：将小于 pivot 的元素放左边，大于的放右边
3. 对左右子数组递归执行上述步骤
4. 终止条件：子数组长度 <= 1
```

#### 分区实现（Lomuto vs Hoare）

**Lomuto 分区（易懂，但效率略低）**：
```
partition(arr, low, high):
    pivot = arr[high]
    i = low - 1          // i 指向"小于区的最后一个"
    for j from low to high-1:
        if arr[j] <= pivot:
            i++
            swap(arr[i], arr[j])
    swap(arr[i+1], arr[high])
    return i+1            // 返回 pivot 最终位置
```

**Hoare 分区（更快，交换次数更少）**：
```
partition(arr, low, high):
    pivot = arr[low + (high-low)/2]  // 通常取中间元素
    i = low - 1
    j = high + 1
    while True:
        do i++ while arr[i] < pivot
        do j-- while arr[j] > pivot
        if i >= j: return j
        swap(arr[i], arr[j])
```

#### 时间复杂度详解

| 情况 | 复杂度 | 发生条件 |
|------|--------|---------|
| 最好 | O(n log n) | 每次 partition 都能将数组均匀二分 |
| 平均 | O(n log n) | 随机数据 |
| 最坏 | O(n^2) | 数组已有序或逆序，且每次选第一个/最后一个作 pivot |

**为什么是 O(n log n)？**
- partition 遍历一次数组，O(n)
- 理想情况下递归深度为 log n
- 总时间 = O(n) * O(log n) = O(n log n)

**最坏情况为什么会发生？**
- 已排序数组 + 选最左/最右作 pivot
- 每次 partition 只分出 1 个和 n-1 个，递归树变成一条链，深度 O(n)
- 总时间 = O(n) * O(n) = O(n^2)

**如何避免最坏情况？**
- 随机选择 pivot（Randomized Quicksort）
- 三数取中法（median-of-three）：选左、中、右三个元素的中位数

#### 空间复杂度

- **O(log n)**：递归调用栈的深度
- 最好情况 O(log n)，最坏情况 O(n)
- 属于**原地排序**（in-place），但递归栈占用额外空间

#### 稳定性

- **不稳定**
- 举例：`[3a, 3b, 1]`，选最后一个 1 作 pivot，3a 和 3b 的相对顺序可能改变

#### 优化技巧

1. **随机 pivot**：避免最坏情况
2. **三数取中**：进一步保证分区均衡
3. **切换插入排序**：当子数组长度小于某个阈值（如 10-20）时，改用插入排序
4. **三路快排**：处理大量重复元素时，将数组分为 < pivot、= pivot、> pivot 三部分

#### 代码模板

```python
def quick_sort(arr, low, high):
    if low < high:
        pi = partition(arr, low, high)
        quick_sort(arr, low, pi - 1)
        quick_sort(arr, pi + 1, high)

# Lomuto 分区
def partition(arr, low, high):
    pivot = arr[high]          # 选最后一个元素
    i = low - 1                # 小于区的边界
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1
```

#### 适用场景

- **通用排序：大多数编程语言的标准库排序实现（C 的 qsort、Python 的 TimSort 也融合了快排思想）**
- **数据量大的随机数据**
- **对稳定性无要求的场景**
- **对空间敏感的场景**（原地排序，额外空间少）

### 2.2 归并排序 (Merge Sort)

#### 核心思想

**分治法**：将数组递归地分成两半，分别排序后，再将两个有序子数组合并成一个有序数组。

#### 算法步骤

```
1. 分解：将数组从中间分成左右两个子数组
2. 递归求解：对左右子数组分别排序
3. 合并：将两个有序子数组合并成一个有序数组
```

#### 合并过程详解

```
合并两个有序数组 [1, 4, 7] 和 [2, 5, 8]：

  i=0          j=0
  ↓            ↓
 [1, 4, 7]    [2, 5, 8]    temp = []

比较 1 和 2，1 < 2，取 1，i++：
  i=1          j=0
  ↓            ↓
 [1, 4, 7]    [2, 5, 8]    temp = [1]

比较 4 和 2，2 < 4，取 2，j++：
  i=1          j=1
  ↓            ↓
 [1, 4, 7]    [2, 5, 8]    temp = [1, 2]

比较 4 和 5，4 < 5，取 4，i++：
  ...依此类推，直到一个数组耗尽，将另一个数组剩余部分全部复制
```

#### 时间复杂度详解

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| 最好 | O(n log n) | 无论数据分布，总是二分 |
| 平均 | O(n log n) | 始终如一 |
| 最坏 | O(n log n) | 稳定不变 |

**为什么始终是 O(n log n)？**
- 递推公式：`T(n) = 2T(n/2) + O(n)`
- 递归树每层处理 O(n)，共 log n 层
- 总时间 = O(n log n)
- **无论数据如何，归并排序的时间复杂度永远不变**，这是它最大的特点

#### 空间复杂度

- **O(n)**：需要额外的数组来存储合并结果
- 递归栈深度 O(log n)，但主要空间消耗在合并数组上
- 这不是原地排序

#### 稳定性

- **稳定**
- 合并时，当左右两个元素相等，优先取左半部分的元素，保证原顺序不变
- `if (left[i] <= right[j])` 而非 `<`

#### 归并排序的变体

1. **自底向上（迭代）归并**：先合并长度为 1 的子数组，然后 2、4、8...
2. **原地归并**：空间复杂度降为 O(1)，但时间会退化
3. **多路归并**：将数组分成 k 路而不是 2 路，用于外部排序

#### 代码模板

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    # 分解
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    # 合并
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:   # <= 保证稳定性
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    # 复制剩余元素
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

#### 适用场景

- **对稳定性有要求的场景**（如按时间排序的交易记录）
- **数据量大的外部排序**（磁盘文件排序）：归并排序天然适合外部存储，可以分块读取
- **链表排序**：归并排序对链表友好，不需要随机访问
- **对最坏情况有严格要求的系统**（如实时系统）

### 2.3 堆排序 (Heap Sort)

#### 核心思想

利用**最大堆（Max-Heap）**这种数据结构。最大堆保证堆顶元素是最大值，每次取出堆顶放到数组末尾，然后调整剩余元素使其重新满足堆性质，反复执行即得到有序序列。

#### 前置知识：堆

- 堆是一棵**完全二叉树**
- **最大堆**：每个父节点的值 >= 子节点的值（根节点最大）
- 用数组存储时：
  - 父节点索引 i 的左子节点：`2*i + 1`
  - 父节点索引 i 的右子节点：`2*i + 2`
  - 子节点索引 j 的父节点：`(j-1) // 2`

#### 算法步骤

```
1. 建堆（Build Max-Heap）：将无序数组调整为最大堆
2. 排序：
   a. 交换堆顶（最大值）与数组末尾元素
   b. 堆的大小减 1（排除已排序的末尾元素）
   c. 对新的堆顶执行下沉（sift-down）操作，恢复堆性质
   d. 重复 a-c 直到堆为空
```

#### 建堆过程详解

```
建堆：从最后一个非叶子节点开始，自底向上执行下沉操作

以数组 [4, 10, 3, 5, 1] 为例：

         4
       /   \
      10    3
     /  \
    5    1

从最后一个非叶子节点（索引 = n//2 - 1 = 1，即值为 10 的节点）开始下沉：

Step 1: 节点 10（索引 1），子节点 5 和 1，10 >= 5 和 1，无需操作
Step 2: 节点 4（索引 0），子节点 10 和 3，最大子节点 10 > 4，交换：

         10
       /    \
      4      3
     / \
    5   1

Step 3: 节点 4（索引 1），子节点 5 和 1，5 > 4，交换：

         10
       /    \
      5      3
     / \
    4   1

堆构建完成！
```

#### 排序过程详解

```
初始最大堆：[10, 5, 3, 4, 1]

Step 1: 交换堆顶 10 与末尾 1 → [1, 5, 3, 4 | 10]，下沉 1 → [5, 4, 3, 1 | 10]
Step 2: 交换堆顶 5 与末尾 1 → [1, 4, 3 | 5, 10]，下沉 1 → [4, 1, 3 | 5, 10]
Step 3: 交换堆顶 4 与末尾 3 → [3, 1 | 4, 5, 10]，下沉 3 → [3, 1 | 4, 5, 10]（无需交换）
Step 4: 交换堆顶 3 与末尾 1 → [1 | 3, 4, 5, 10]，下沉...但堆大小为 1，结束

最终结果：[1, 3, 4, 5, 10]
```

#### 时间复杂度详解

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| 最好 | O(n log n) | 所有情况一致 |
| 平均 | O(n log n) | 稳定不变 |
| 最坏 | O(n log n) | 稳定不变 |

**为什么始终是 O(n log n)？**
- 建堆：O(n) —— 对 n/2 个节点执行下沉，每次下沉 O(log n) 但大部分节点下沉次数少，摊还分析得到 O(n)
- 排序：n-1 次交换和下沉，每次下沉 O(log n)，共 O(n log n)
- 总时间：O(n) + O(n log n) = **O(n log n)**

**为什么堆排序实际比快排慢？**
- **缓存不友好**：堆排序的数组访问模式是跳跃式的（父节点到子节点），缺乏空间局部性
- **交换次数多**：建堆和排序过程中有大量交换操作
- 快速排序是顺序访问，CPU 缓存命中率高

#### 空间复杂度

- **O(1)**：原地排序，仅需常数级额外空间
- 这是堆排序相对于归并排序的最大优势

#### 稳定性

- **不稳定**
- 建堆和交换过程中，相同元素的相对顺序无法保持
- 例：`[5a, 5b, 3]` 建堆后 5a 和 5b 的相对顺序可能改变

#### 代码模板

```python
def heap_sort(arr):
    n = len(arr)

    # 建堆
    for i in range(n // 2 - 1, -1, -1):
        sift_down(arr, n, i)

    # 排序
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # 堆顶与末尾交换
        sift_down(arr, i, 0)              # 对新的堆顶下沉

def sift_down(arr, n, i):
    """在大小为 n 的堆中，对索引 i 执行下沉操作"""
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2

    if left < n and arr[left] > arr[largest]:
        largest = left
    if right < n and arr[right] > arr[largest]:
        largest = right

    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        sift_down(arr, n, largest)  # 递归下沉
```

#### 适用场景

- **对空间极度敏感的场景**（O(1) 额外空间）
- **嵌入式系统 / 资源受限环境**
- **需要稳定的 O(n log n) 最坏时间**（快排在极端情况会退化）
- **Top K 问题**：用大小为 K 的最小堆可以在 O(n log K) 时间内找到前 K 大元素
- **优先级队列、任务调度**（堆本身就是优先级队列的经典实现）
- **数据流中的中位数**：用最大堆 + 最小堆配合

## 3. 横向对比表格

| 维度 | 快速排序 | 归并排序 | 堆排序 |
|------|---------|---------|--------|
| 平均时间复杂度 | O(n log n) | O(n log n) | O(n log n) |
| 最好时间复杂度 | O(n log n) | O(n log n) | O(n log n) |
| 最坏时间复杂度 | O(n^2) | O(n log n) | O(n log n) |
| 空间复杂度 | O(log n) | O(n) | O(1) |
| 是否稳定 | 否 | **是** | 否 |
| 是否原地排序 | 是（忽略递归栈） | 否 | 是 |
| 最坏情况 | 已有序 + 糟糕 pivot 选择 | 无 | 无 |
| 对缓存友好度 | **高**（顺序访问） | 中（需要额外数组） | **低**（跳跃访问） |
| 适合链表 | 否（需随机访问） | **是** | 否（需随机访问） |
| 适合外部排序 | 否 | **是** | 否 |
| 实现难度 | 中 | 低 | 中高 |
| 实际表现 | **最快**（一般情况） | 中 | 较慢 |

## 4. 适用场景决策指南

### 场景一：通用排序

```
优先选 快速排序
  ├─ 数据随机？         → 快速排序（默认最优）
  ├─ 数据接近有序？     → 快速排序（随机 pivot 优化）
  ├─ 有大量重复元素？   → 三路快排
  └─ 非常在意最坏情况？ → 归并排序 或 堆排序
```

### 场景二：对稳定性有要求

```
优先选 归并排序
  ├─ 数据在内存中？     → 归并排序
  ├─ 数据在磁盘上？     → 归并排序（外部排序）
  └─ 需要 O(1) 空间？   → 稳定排序极少原地，可使用插入排序（小数据）
```

### 场景三：空间受限

```
优先选 堆排序
  └─ 嵌入式/低内存设备？ → 堆排序（O(1) 空间）
```

### 场景四：链表排序

```
优先选 归并排序
  └─ 单向/双向链表？    → 归并排序（无需额外空间分配，只需修改指针）
```

### 场景五：Top K / 优先级问题

```
优先选 堆
  ├─ 求前 K 大/小元素   → 堆（O(n log K)）
  ├─ 数据流中位数       → 最大堆 + 最小堆
  └─ 任务调度           → 堆（优先级队列）
```

### 决策流程图

```
需要排序？
  │
  ├─ 数据在外部存储（磁盘/网络）？ → 归并排序
  │
  ├─ 对稳定性有要求？
  │   ├─ 是 → 归并排序
  │   └─ 否 → ↓
  │
  ├─ 空间极度受限（O(1)）？
  │   ├─ 是 → 堆排序
  │   └─ 否 → ↓
  │
  ├─ 需要最坏情况也有保障？
  │   ├─ 是 → 归并排序 或 堆排序
  │   └─ 否 → ↓
  │
  └─ 默认选择 → 快速排序（最快、原地、实现灵活）
```

## 7. 参考答案与解析

### 基础题答案

**A1**:
最坏时间复杂度为 **O(n^2)**。发生在**每次分区都极不平衡**时，例如：
- 数组已经**有序**且每次选择第一个或最后一个元素作为 pivot
- 每次 partition 只分出 1 个和 n-1 个元素
- 递归深度从 log n 退化为 n，总时间从 O(n log n) 退化为 O(n^2)
- **解决方案**：随机选择 pivot 或使用三数取中法

**A2**:
空间复杂度为 **O(n)**。原因：
- 合并阶段需要**额外数组**来临时存储两个有序子数组的合并结果
- 虽然递归栈深度为 O(log n)，但合并过程需要 O(n) 的辅助空间，取最大值
- 如果使用自底向上（迭代）的归并排序，可以避免递归栈，但合并数组的空间仍是 O(n)

**A3**:
建堆的时间复杂度是 **O(n)**，不是 O(n log n)。

粗略看，会对 n/2 个节点执行下沉，每个下沉最多 O(log n)，似乎是 O(n log n)。但更精确的分析是：

**为什么是 O(n)？**

堆是一棵完全二叉树。设树高为 h，则：
- 第 k 层（从下往上数，k=1 是最底层）有 n/2^k 个节点
- 第 k 层的节点最多下沉 k 步
- 总操作数 = sum_{k=1}^{h} (n/2^k) * k = n * sum_{k=1}^{h} k/2^k <= n * 2 = O(n)

也就是说，**大多数节点位于底层，下沉距离短**，摊还下来整体是 O(n)。

**A4**:
稳定的排序算法：**B. 归并排序**、**D. 插入排序**

补充：
- 冒泡排序：稳定
- 归并排序：稳定（关键在合并时用 <=）
- 插入排序：稳定
- 计数排序、桶排序、基数排序：稳定
- 快速排序：不稳定
- 选择排序：不稳定
- 堆排序：不稳定
- 希尔排序：不稳定

**A5**:
**插入排序**。当数据近乎有序时，插入排序的时间复杂度接近 O(n)。

虽然快速排序通常很快，但处理近乎有序的数据时，如果 pivot 选得不好（选第一个/最后一个），反而会触发最坏情况 O(n^2)。而插入排序对于近乎有序的数据，每个元素只需比较几次就能找到正确位置，整体接近线性。

很多实际排序库（如 Python 的 TimSort）会将插入排序和归并排序结合，利用插入排序处理小规模/有序片段的优势。

**A6**:
最后一个非叶子节点的索引是 **n // 2 - 1**（0-based index）。

证明：完全二叉树中，最后一个节点索引为 n-1，其父节点为最后一个非叶子节点。
父节点索引 = (n - 1 - 1) // 2 = (n - 2) // 2 = n // 2 - 1。

### 进阶题答案

**A7**:
原因主要有以下几点：

1. **缓存不友好（Cache Unfriendliness）**
   - 快速排序访问模式是**顺序的**：partition 过程从左到右扫描数组，CPU 的缓存预取机制可以很好利用空间局部性
   - 堆排序访问模式是**跳跃的**：下沉操作需要在父节点和子节点之间跳转（索引 i -> 2i+1 或 2i+2），尤其是堆底部的节点，访问跨度很大，缓存命中率低

2. **交换次数**
   - 堆排序在建堆和排序阶段有大量交换操作
   - 快速排序的交换次数相对较少

3. **常数因子**
   - 时间复杂度的 O(n log n) 忽略了常数因子
   - 快速排序的常数因子远小于堆排序的实际常数（约 2-3 倍差距）

4. **数据移动距离**
   - 堆排序中，一个元素可能被移动 O(log n) 次，而且每次都通过交换
   - 快速排序中，partition 过程类似于将元素"拨"到正确的一侧，移动距离总和为 O(n)

**A8**:
使用 **归并排序的外部排序版本**（External Merge Sort）。

具体方案：
1. **分阶段**：将 100GB 文件分成 25 个约 4GB 的块（留出内存给程序运行）
2. **排序阶段**：每次读入一个块到内存，用快速排序或归并排序排序后写回磁盘
3. **合并阶段**：使用多路归并（如 25 路归并），每次从每个文件中读一个元素，选出最小的输出
4. 重复合并过程，直到所有记录归为一个文件

**为什么选归并排序？**
- 归并排序天然支持**顺序读写磁盘**（磁盘的随机 I/O 极慢）
- 可以很好地控制内存使用
- 快排和堆排序不适合外部排序，因为它们需要随机访问数据

**A9**:
**三路快排原理**：

普通快排将数组分为两部分：< pivot 和 >= pivot。
三路快排将数组分为三部分：< pivot、= pivot、> pivot。

```
分区前：[ < pivot | = pivot | 未知区域 | > pivot ]
                 ↑         ↑         ↑
                lt        i         gt
```

- `[low, lt-1]`：小于 pivot 的区域
- `[lt, gt]`：等于 pivot 的区域
- `[gt+1, high]`：大于 pivot 的区域
- 用三个指针 lt（less than）、i（当前扫描位置）、gt（greater than）来维护

**解决的问题**：
当数组中有**大量重复元素**时，普通快排会把相等的元素分散到两边，导致分区严重不平衡，退化为 O(n^2)。三路快排把所有等于 pivot 的元素集中在中间，左右两边只需要处理真正小于和大于的元素，即使所有元素都相等，也只需一次 O(n) 扫描就结束。

荷兰国旗问题（Dutch National Flag Problem）就是三路快排的经典原型。

**A10**:
**计数排序**（Counting Sort）。

原因：
- 只有 10 种不同的值，说明计数排序需要的辅助数组大小就是 10
- 计数排序的时间复杂度为 O(n + k) = O(n + 10) = O(n)
- 当 k 远小于 n 时，计数排序是线性时间，远快于 O(n log n) 的比较排序

如果这 10 种值分布不均匀（比如大量集中在少数值上），桶排序也是不错选择。

**A11**:
**方法**：使用一个**大小为 K 的最大堆**。

步骤：
1. 用数组前 K 个元素建一个最大堆
2. 遍历剩余元素（从索引 K 到 n-1）：
   - 如果当前元素小于堆顶（堆中最大元素），则弹出堆顶，将当前元素入堆
   - 否则跳过
3. 遍历结束后，堆中就是最小的 K 个数

**时间复杂度**：O(n log K)
- 建堆：O(K)
- 每次操作（可能入堆）：O(log K)
- 总共遍历 n 个元素，最坏情况每个都入堆：O(n log K)

**注意**：如果 K 接近 n，不如直接用快速排序取前 K 个。

**A12**:
**不完全准确**，需要分情况讨论：

**严格来说不是原地排序**：
- 快速排序需要**递归调用栈**，栈深度平均 O(log n)，最坏 O(n)
- 原地排序要求额外空间为 O(1)

**但通常被称为"原地排序"**：
- 很多教材和资料将快速排序归类为原地排序，因为它**不需要额外的数据存储结构**（如归并排序的辅助数组）
- 它通过在原数组上进行交换操作完成排序
- 递归栈通常不被视为"额外数据空间"

**准确说法**：
- 快速排序是**半原地排序**或**就地排序**（排序在原数组上进行，但递归栈占用额外 O(log n) 空间）
- 严格意义上的原地排序只有堆排序、插入排序等

**A13**: 寻找第 K 大元素（基于快速选择的 QuickSelect）

```python
import random

def findKthLargest(nums, k):
    """
    寻找数组中第 K 大的元素（基于快速选择）
    平均时间复杂度 O(n)，最坏 O(n^2)
    """

    def partition(left, right):
        """Lomuto 分区，返回 pivot 最终位置"""
        # 随机选 pivot 避免最坏情况
        pivot_idx = random.randint(left, right)
        nums[pivot_idx], nums[right] = nums[right], nums[pivot_idx]

        pivot = nums[right]
        i = left
        for j in range(left, right):
            if nums[j] >= pivot:  # 注意：第 K 大，所以 >= 放左边
                nums[i], nums[j] = nums[j], nums[i]
                i += 1
        nums[i], nums[right] = nums[right], nums[i]
        return i

    left, right = 0, len(nums) - 1
    target = k - 1  # 第 K 大在 0-based 中的索引

    while left <= right:
        pos = partition(left, right)
        if pos == target:
            return nums[pos]
        elif pos < target:
            left = pos + 1
        else:
            right = pos - 1
    return -1


# 测试
print(findKthLargest([3, 2, 1, 5, 6, 4], 2))   # 输出: 5
print(findKthLargest([3, 2, 3, 1, 2, 4, 5, 5, 6], 4))  # 输出: 4
```

**A14**: 链表排序（归并排序）

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def sortList(head):
    """
    对链表排序，要求 O(n log n) 时间，O(1) 额外空间（递归栈不计）
    使用归并排序：链表归并不需要额外数组，只需修改指针
    """

    if not head or not head.next:
        return head

    # 1. 找中点（快慢指针）
    slow, fast = head, head.next
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    mid = slow.next
    slow.next = None  # 断开链表

    # 2. 递归排序左右
    left = sortList(head)
    right = sortList(mid)

    # 3. 合并两个有序链表
    dummy = ListNode(0)
    curr = dummy
    while left and right:
        if left.val <= right.val:  # 稳定排序
            curr.next = left
            left = left.next
        else:
            curr.next = right
            right = right.next
        curr = curr.next
    curr.next = left or right

    return dummy.next
```

**A15**: 合并两个有序数组（原地，不使用额外数组）

```python
def merge(nums1, m, nums2, n):
    """
    合并两个有序数组 nums1 和 nums2
    nums1 的长度为 m+n，前 m 个是有效元素
    要求在 nums1 上原地合并，不使用额外数组
    时间复杂度 O(m+n)，空间复杂度 O(1)
    """
    # 三个指针：从后往前遍历
    p1 = m - 1      # nums1 有效元素的末尾
    p2 = n - 1      # nums2 的末尾
    p = m + n - 1   # 合并后数组的末尾

    while p1 >= 0 and p2 >= 0:
        if nums1[p1] >= nums2[p2]:
            nums1[p] = nums1[p1]
            p1 -= 1
        else:
            nums1[p] = nums2[p2]
            p2 -= 1
        p -= 1

    # 如果 nums2 还有剩余，全部复制到 nums1 前部
    # 如果 nums1 还有剩余，它已经在正确位置，不需要处理
    nums1[:p2 + 1] = nums2[:p2 + 1]


# 测试
nums1 = [1, 2, 3, 0, 0, 0]
merge(nums1, 3, [2, 5, 6], 3)
print(nums1)  # 输出: [1, 2, 2, 3, 5, 6]
```

**A16**: 数组元素在 [0, n-1] 范围内的排序

```python
def sort_special(nums):
    """
    数组元素范围在 [0, n-1] 之间（可能有重复）
    要求 O(n) 时间，O(1) 额外空间
    思路：利用元素值作为索引，将元素放到正确位置
    """
    n = len(nums)
    i = 0
    while i < n:
        # nums[i] 应该在索引 nums[i] 处
        # 如果不在正确位置上，且目标位置不是正确的（避免死循环）
        if nums[i] != i:
            # 将 nums[i] 放到它应该在的位置
            target = nums[i]
            # 如果目标位置已经有正确的值，跳过
            if nums[target] == target:
                # 这个值是重复的，跳过
                i += 1
            else:
                nums[i], nums[target] = nums[target], nums[i]
        else:
            i += 1
    return nums


# 测试
print(sort_special([2, 3, 1, 0, 3, 5, 4]))
# 输出: [0, 1, 2, 3, 3, 4, 5]
```

**时间复杂度**：O(n) —— 每个元素最多被交换一次到正确位置。
**空间复杂度**：O(1) —— 原地交换，没有使用额外数组。
