## 结构化笔记

### 一、三大排序概览对比

| 特性 | 快速排序 | 归并排序 | 堆排序 |
|------|---------|---------|-------|
| **平均时间复杂度** | O(n log n) | O(n log n) | O(n log n) |
| **最坏时间复杂度** | O(n^2) | O(n log n) | O(n log n) |
| **最好时间复杂度** | O(n log n) | O(n log n) | O(n log n) |
| **空间复杂度** | O(log n) ~ O(n) | O(n) | O(1) |
| **稳定性** | 不稳定 | **稳定** | 不稳定 |
| **是否基于比较** | 是 | 是 | 是 |
| **排序方式** | 内部排序（原地） | 内外均可 | 内部排序（原地） |

- **一句话解释**：三者都是基于"分治"或"选择"思想的高级排序算法，平均时间复杂度都是 O(n log n)，但在空间、稳定性、最坏情况表现上差异很大。
- **易错点/考点提示**：
  - 快速排序的**最坏情况**发生在每次选的 pivot 都是最大或最小元素（如数组已有序），此时退化为 O(n^2)，优化手段包括随机选 pivot 或三数取中法。
  - 归并排序是 O(n log n) 级别中**唯一稳定**的排序算法（408 高频选择题考点）。
  - 堆排序的"建堆"操作时间复杂度是 O(n)，不是 O(n log n) —— 这是一个常见误解。
- **与已学知识的关联**：三种排序都体现了重要的算法设计思想——快速排序体现了"随机化+分治"，归并排序体现了"分而治之+合并"，堆排序体现了"数据结构选择决定算法效率"（用堆这种数据结构来优化选择过程）。

### 二、快速排序（Quick Sort）

#### 是什么
快速排序是一种基于"分治"思想的排序算法。核心操作是**划分（partition）**：选定一个基准元素（pivot），将数组分为两部分，左边都比 pivot 小，右边都比 pivot 大，然后递归地对左右两部分排序。

#### 关键要点
- **划分函数**是快速排序的核心，408 常考手动模拟一趟划分的过程。
- **pivot 的选择**直接影响性能：固定选第一个/最后一个 → 有序数组时最坏；随机选 / 三数取中 → 避免最坏情况。
- **空间复杂度**：递归调用需要栈空间，平均 O(log n)，最坏 O(n)。

#### 适用场景
- 大规模无序数据排序，平均性能最优。
- 编程语言内置排序的首选实现（如 C 的 qsort、Java 的 `Arrays.sort()` 对基本类型的排序）。
- **不适合**：数据基本有序或数据量极小（小规模不如插入排序）。

#### 易错点
- 划分时 left 和 right 指针的移动顺序（先右后左还是先左后右）会影响结果。
- 快速排序是**不稳定**的，因为 pivot 的交换会打乱相等元素的相对顺序。
- 递归深度问题：最坏情况下递归深度 O(n)，有栈溢出风险。

---

### 三、归并排序（Merge Sort）

#### 是什么
归并排序也是一种"分治"算法，但它的核心思想是**先分解，再合并**：把数组递归地分成两半，直到每组只有一个元素（天然有序），然后两两合并成有序序列。

#### 关键要点
- **合并操作**是归并排序的核心，需要额外的辅助数组来存储合并结果。
- **二路归并**是最常见的形式，也可以推广到多路归并，用于外部排序。
- **时间复杂度稳定**：无论数据分布如何，都是 O(n log n)，没有最坏情况。

#### 适用场景
- **需要稳定排序**时（如按关键字排序）。
- **链表排序**：归并排序对链表很友好，不需要额外空间（链表合并不需要额外数组）。
- **外部排序**：当数据量太大，无法全部装入内存时，归并排序是基础（多路归并 + 败者树）。
- **不适合**：对空间敏感的场景（需要 O(n) 额外空间）。

#### 易错点
- 归并排序的空间复杂度是 O(n) —— 需要额外的数组来合并，递归栈 O(log n) 通常不计入或认为是 O(n + log n) = O(n)。
- **它是稳定排序**的原因：合并时遇到相等元素，优先将左子序列的元素放入结果数组。
- 408 中常考归并排序的**趟数**：对 n 个元素做二路归并，需要 ⌈log₂n⌉ 趟。

---

### 四、堆排序（Heap Sort）

#### 是什么
堆排序利用**堆**这种数据结构来完成排序。堆是一棵完全二叉树，分为大根堆（父节点 >= 子节点）和小根堆（父节点 <= 子节点）。排序过程：先建堆，然后反复将堆顶元素（最大/最小）与堆末元素交换，再调整堆。

#### 关键要点
- **建堆**：从最后一个非叶子节点往前依次做"向下调整"（sift-down），时间为 O(n)。
- **排序**：每次取堆顶 swap 到末尾，然后对剩余元素做调整，共 n-1 次，每次 O(log n)，总计 O(n log n)。
- **空间复杂度 O(1)**：完全原地排序，只需常数级额外空间。

#### 适用场景
- **Top-K 问题**：用大小为 K 的小根堆，遍历一遍数据即可找出前 K 大的元素。
- **内存受限环境**：不需要额外数组，空间效率极高。
- **优先级队列**：堆是实现优先队列的标准数据结构。
- **不适合**：需要稳定排序时；对缓存不友好（跳跃式访问数组，不如快速排序的局部性访问好）。

#### 易错点
- 建堆的复杂度是 O(n)，不是 O(n log n) —— 从下往上调整，大部分节点深度浅。
- 堆排序是**不稳定**的，因为堆顶和堆末的交换会破坏稳定性。
- **堆的向下调整 vs 向上调整**：建堆用向下调整，插入节点用向上调整。
- 408 常考：给定一个序列，手动模拟建堆过程和每一趟排序的结果。

### 五、稳定性深入辨析

| 排序算法 | 稳定？ | 为什么？ |
|---------|-------|---------|
| 快速排序 | 不稳定 | pivot 交换时可能将相等的元素换到另一侧 |
| 归并排序 | **稳定** | 合并时遇到相等元素优先取左子序列 |
| 堆排序 | 不稳定 | 堆顶与堆末交换时可能打乱相等元素的顺序 |
| 冒泡排序 | 稳定 | 相邻交换，相等不交换 |
| 插入排序 | 稳定 | 向前插入，相等不移动 |
| 选择排序 | 不稳定 | 选最小与前面交换可能打乱顺序 |
| 希尔排序 | 不稳定 | 分组后跨越式插入 |
| 基数排序 | 稳定 | 按位分配收集，先进先出 |

- **一句话记忆**：在 O(n log n) 级别的排序中，只有**归并排序**是稳定的。

---

### 六、关联思考

- **排序算法的选择 = 时间-空间-稳定性-数据特征的权衡**：
  - 不在乎稳定性、数据随机 → 快速排序（最快）
  - 在乎稳定性、不在乎空间 → 归并排序
  - 在乎空间、不在乎稳定性 → 堆排序

- **与操作系统/计组的交叉**：
  - 快速排序的递归栈 → 函数调用栈的深度与内存管理相关
  - 归并排序的多路归并 → 操作系统的外部排序，涉及磁盘 I/O 优化（如磁盘调度算法）
  - 堆排序的完全二叉树 → 可以用数组存储，即隐式数据结构，和计组中 Cache 的连续性相关

- **与数据结构的交叉**：
  - 堆排序的"堆"就是数据结构中的**优先队列**
  - 归并排序的"合并有序子序列"思想在**合并 K 个有序链表**问题中出现

## 费曼讲解

### 费曼讲解：快速排序

> 快速排序就像给一堆试卷按分数排序。你先随便抽一张试卷（选 pivot），然后让所有分数比它低的放左边，比它高的放右边。现在你手上这三堆（低、中、高）就相对有序了。接着对"低"这堆和"高"这堆分别再重复同样的操作，直到每堆只剩一张试卷就排完了。
>
> **常见误解**：很多人以为快速排序总是很快。但它最怕数组已经排好序——如果你每次都抽到第一个作为基准，那就相当于每次只分出一个元素，退化成冒泡排序 O(n^2)。
>
> **一句话记忆点**: 选个基准，小的放左大的放右，递归分治，小心有序数组！

### 费曼讲解：归并排序

> 归并排序也可以拿试卷排序来类比，但方法不同。你先不管三七二十一，把一堆试卷从中间切成两半，每半再切两半……一直切到每份试卷单独一摞（只有一个元素，天然有序）。现在开始合并：把两个有序的小摞合并成一个有序的大摞，就像洗两副已经排好序的扑克牌，一张一张比较大小放好。不断合并，直到变回一整摞有序的试卷。
>
> **常见误解**：有人以为归并排序不需要额外空间。实际上合并时你需要一张空桌子（额外数组）来放合并结果。这也是它空间复杂度 O(n) 的原因。
>
> **一句话记忆点**: 先切到最小，再两两合并，稳定有序，就是费点空间！

### 费曼讲解：堆排序

> 堆排序的思路更像是"每次选出最大的"。想象你要整理一堆数字卡片。你先花点时间，把它们堆成一个"金字塔"（大根堆），塔尖上永远是最大的那张卡片。然后你把塔尖拿走（放到末尾），再把剩下的重新调整成新的金字塔，继续取塔尖……直到取完。
>
> **常见误解**：很多人以为建堆要花 O(n log n)，其实只花 O(n)。原因是——大部分节点都在堆的底部，调整它们只需要几步，平均值算下来是线性的。而且堆排序全程就在原来的卡片堆上操作，不需要额外空间。
>
> **一句话记忆点**: 把数组堆成金字塔，每次取塔尖放末尾，不占空间，但稳定性差！

### 理解辨析

**第 3 题**
> 快速排序在什么情况下会退化为 O(n^2)？如何优化？

<details>
<summary>点击查看答案</summary>

**答案**: 当每次划分选取的 pivot 都是当前子数组的最大值或最小值时（例如数组已经有序或逆序，且固定选择第一个或最后一个元素作为 pivot），快速排序会退化为 O(n^2)。

**优化方法**：
1. **随机选 pivot**：随机选取一个元素作为 pivot，从概率上避免最坏情况。
2. **三数取中法**：选取左端、右端、中间三个元素的中位数作为 pivot。
3. **小规模改用插入排序**：当子数组长度小于某个阈值（如 10-15）时，改用插入排序。
4. **三路划分**：将数组分为小于、等于、大于 pivot 三部分，处理大量重复元素时效果更好。
</details>

**第 4 题**
> 判断：堆排序的建堆过程时间复杂度是 O(n log n)。这种说法对吗？请说明理由。

<details>
<summary>点击查看答案</summary>

**答案**: 不对。建堆的时间复杂度是 **O(n)**，不是 O(n log n)。

**解析**: 建堆是从最后一个非叶子节点开始，依次对每个节点执行向下调整（sift-down）。关键点在于——堆中高度为 h 的节点最多有 ⌈n/2^(h+1)⌉ 个，向下调整的时间与高度 h 成正比。总时间 = Σ(h * 该高度节点数) ≈ O(n)。严格推导得出建堆是线性时间复杂度。
</details>

**第 5 题**
> 为什么归并排序是稳定的，而快速排序和堆排序是不稳定的？请分别用一句话说明原因。

<details>
<summary>点击查看答案</summary>

**答案**:
- **归并排序稳定**：合并两个有序子序列时，遇到相等元素，总是优先取左子序列的元素，保证了相等元素的原始相对顺序不变。
- **快速排序不稳定**：划分时 pivot 的交换会跨越多个位置，可能导致相等元素的相对顺序被翻转。
- **堆排序不稳定**：堆顶元素与堆末元素交换时，可能将后面出现的相等元素换到前面去。

**一句话记忆**: 归并的有序合并保顺序，快排的跨越交换乱顺序，堆排的首尾置换也乱顺序。
</details>

### 综合应用

**第 6 题**
> 给定数组 `[3, 7, 1, 9, 4, 6, 8, 2]`，使用快速排序进行排序，请写出以第一个元素为 pivot、进行**第一趟划分**后的数组结果。

<details>
<summary>点击查看答案</summary>

**答案**: `[2, 1, 3, 9, 4, 6, 8, 7]`（答案可能因划分算法的具体实现略有不同）

**解析**: 选 pivot = 3。使用经典的 Hoare 或 Lomuto 划分：

以 Lomuto 为例（这里采用常见的双指针法）：
1. i 指向左端（索引0），j 指向右端（索引7），pivot = 3
2. i 从左向右找 >= pivot 的元素，停在 3（索引0）
3. j 从右向左找 <= pivot 的元素，停在 2（索引7）
4. 交换 3 和 2 → `[2, 7, 1, 9, 4, 6, 8, 3]`
5. i 右移找 >= pivot：7（索引1）；j 左移找 <= pivot：3（索引7）
6. 交换 7 和 3 → `[2, 3, 1, 9, 4, 6, 8, 7]`
7. i 右移找 >= pivot：3（索引1）；j 左移找 <= pivot：1（索引2）→ 此时 i > j，停止
8. 将 pivot 放到最终位置（索引1）：交换索引1和索引0 → pivot 3 的位置在索引1

一趟划分后：pivot = 3 已就位，左边 `[2, 1]` 都 <= 3，右边 `[9, 4, 6, 8, 7]` 都 >= 3。

最终结果：`[2, 1, 3, 9, 4, 6, 8, 7]`
</details>

**第 7 题**
> 在含有 n 个元素的大根堆中，删除堆顶元素（即最大值）后，如何调整使其重新成为大根堆？请结合以下数组说明过程。
>
> 大根堆（数组表示）：`[9, 7, 8, 5, 3, 6, 4, 1, 2]`

<details>
<summary>点击查看答案</summary>

**答案**: 删除堆顶（即取出 9）后，把堆末元素 2 放到堆顶，然后对堆顶执行「向下调整」：

1. 初始堆（完全二叉树对应数组）：9 在堆顶，删除 9 并将最后一个元素 2 移到堆顶
2. 此时堆数组为：`[2, 7, 8, 5, 3, 6, 4, 1, 2 已被移除]` → 长度为 8：`[2, 7, 8, 5, 3, 6, 4, 1]`
3. 从堆顶（索引0，值=2）开始向下调整：
   - 左孩子索引1（值=7），右孩子索引2（值=8），较大的是右孩子 8
   - 2 < 8，交换 2 和 8 → `[8, 7, 2, 5, 3, 6, 4, 1]`
4. 继续对索引2（值=2）调整：
   - 左孩子索引5（值=6），右孩子索引6（值=4），较大的是左孩子 6
   - 2 < 6，交换 2 和 6 → `[8, 7, 6, 5, 3, 2, 4, 1]`
5. 继续对索引5（值=2）调整：
   - 左孩子索引11（超出数组长度），停止
6. 调整完毕，新的大根堆：`[8, 7, 6, 5, 3, 2, 4, 1]`
</details>

**第 8 题**
> 有一个长度为 100000 的无序数组，要求找出其中最大的 10 个元素。请选择最合适的排序算法，并说明理由。

<details>
<summary>点击查看答案</summary>

**答案**: **堆排序（Top-K 变体）** 最合适。

**解析**:
1. 建立一个大小为 **10 的小根堆**。
2. 遍历数组中的每个元素：
   - 如果元素比堆顶大，则替换堆顶，并向下调整堆。
   - 否则跳过。
3. 遍历结束后，堆中的 10 个元素就是最大的 10 个元素。

**为什么不是其他算法？**
- 快速排序：需要全排序，时间复杂度 O(n log n)，而且需要递归栈空间。
- 归并排序：需要全排序 + O(n) 额外空间。
- 堆排序的 Top-K 变体时间复杂度为 O(n log K)，K=10 时近似 O(n)，空间 O(1)，是最优解。

**关联知识点**: 这个问题是「堆的应用」的经典考题，也是 408 的高频应用题。
</details>

### 关联思考

**第 9 题**
> 外部排序（如磁盘文件排序）为什么通常基于归并排序，而不是快速排序或堆排序？

<details>
<summary>点击查看答案</summary>

**答案**: 外部排序的核心瓶颈是**磁盘 I/O 次数**，而不是 CPU 计算时间。

**原因分析**：
1. **归并排序天然适合外排**：二路归并可以扩展为多路归并，通过增加路数（k 路归并）来减少归并趟数，从而大幅降低磁盘读写次数。结合败者树优化，选择最小元素的时间复杂度从 O(k) 降到 O(log k)。
2. **快速排序不适合**：快速排序的划分操作需要频繁随机访问数组元素，而磁盘的随机访问速度远慢于顺序访问。外部排序中应尽量使用**顺序读写**，归并排序的合并操作正好是顺序访问。
3. **堆排序不适合**：堆排序的 sift-down 操作涉及跳跃式数组访问，对磁盘 Cache 不友好。此外堆排序无法像归并那样通过多路方式扩展以减少 I/O。

**一句话总结**: 外部排序要尽量减少磁盘 I/O，归并排序支持顺序读写和多路扩展，是唯一适合的 O(n log n) 排序。
</details>

**第 10 题**
> 快速排序、归并排序、堆排序，如果让你在面试中手写实现，你选哪个？在实际工程中，语言内置的排序函数（如 Python 的 `sorted()`、Java 的 `Arrays.sort()`、C++ 的 `std::sort()`）分别采用的是什么排序策略？这些选择背后体现了什么设计思想？

<details>
<summary>点击查看答案</summary>

**答案**:

**面试手写推荐：快速排序**
- 代码简洁（核心就划分和递归两部分），容易在面试中展示对分治思想的理解。
- 但需要小心处理 pivot 选择和边界条件。

**各语言内置排序策略**：
- **Python `sorted()` / `.sort()`**：Timsort（归并排序的优化变体 + 插入排序），稳定排序。Python 的设计哲学强调可读性和稳定性。
- **Java `Arrays.sort()`**：
  - 基本类型：Dual-Pivot QuickSort（双轴快速排序），不稳定。
  - 对象类型：Timsort（归并排序变体），稳定。
  - 设计考量：基本类型不需要稳定性，追求速度；对象类型需要稳定性（equals 方法期望）。
- **C++ `std::sort()`**：IntroSort（内省排序），开始时用快速排序，递归深度超标时切换到堆排序，小规模用插入排序。不稳定。C++ 的设计哲学追求极限性能和零开销抽象。

**设计思想的体现**：
1. **没有万能排序**：不同语言、不同数据类型会选择不同的排序策略，反映了「没有银弹」的工程哲学。
2. **混合策略**：现代排序实现都采用混合策略（如 Introsort、Timsort），取各算法之长。
3. **稳定性 vs 性能**：对象/引用类型倾向于稳定排序，基本类型倾向于快速排序。
</details>

## 复习建议

### 发现的薄弱环节

根据排序算法的学习特点，以下是最容易踩坑的环节：

1. **三者之间的横向对比不够清晰**：建议把快速排序、归并排序、堆排序的对比表在纸上默写一遍，特别是空间复杂度和稳定性这两个维度。
2. **手动模拟不够熟练**：408 和面试中经常要求手动模拟一趟划分 / 一次堆调整 / 一次合并，建议在白板上画完全二叉树和数组下标对应关系来练习。
3. **时间复杂度的直觉判断**：尤其是「建堆为什么是 O(n)」和「快速排序最坏情况」这两个点。

### 建议下一步

1. **立刻用白板/纸笔手动模拟**：
   - 快速排序：随机选一个数组，手动做 2-3 趟划分
   - 堆排序：从无序数组开始，手动模拟建堆 + 排序全过程
   - 归并排序：手动模拟合并两个有序数组的过程

2. **代码实现**：尝试在不查资料的情况下手写三种排序的代码，写完再对照标准实现检查边界条件。

3. **进阶学习**：
   - 了解 **Introsort**（C++ STL 使用的排序，融合了快排、堆排、插排）
   - 了解 **Timsort**（Python/Java 对象排序使用的稳定排序，融合了归并和插排）
   - 了解 **外部排序** 中的多路归并和败者树

4. **推荐资源**：
   - 视频：B站/YouTube 上搜索 "Sorting Algorithm Visualized"，看动画演示有助于建立直觉
   - 练习：LeetCode 上刷排序相关题目，如 215（数组中的第 K 个最大元素）、148（排序链表）、912（排序数组）—— 练习用三种排序分别实现
   - 如果准备 408 考试：重点做历年真题中排序相关的选择题和大题

### 鼓励的话

> 排序算法是数据结构的"基本功"，也是面试和考研的"送分题"——只要肯花时间手动模拟几遍，把对比表记牢，这部分几乎是稳拿分的。把这三个吃透了，你不仅掌握了排序，还掌握了分治、堆这样的重要思想，后面学图算法、动态规划都会轻松很多。加油！
